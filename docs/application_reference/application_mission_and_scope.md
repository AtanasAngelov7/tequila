# Application Mission & Product Scope

Last updated: March 12, 2026

## 1) What Tequila is

Tequila is a personal AI agent platform designed for non-technical everyday users. It combines:
- conversational assistance,
- multi-agent execution,
- persistent memory/knowledge,
- workflow automation,
- and multi-channel communication.

The product is intentionally broader than a chatbot: it aims to become a persistent operating layer that can act, remember, and coordinate work over time.

## 2) Primary goal

The core goal is to let a user describe outcomes in natural language and have the system:
1. understand intent,
2. execute safely using tools and integrations,
3. retain useful context for future work,
4. and proactively assist via workflows, schedules, and notifications.

In practical terms, Tequila is targeting "assist + automate + remember" in one coherent product.

## 3) Who it is for

- Users who work heavily with information (notes, email, planning, files) but are not engineers.
- Users who need one assistant experience across web UI and messaging channels.
- Users who want proactive help, not only reactive Q&A.

## 4) Product boundaries

Tequila currently focuses on personal agent operations rather than enterprise replacement tooling.

It does **not** try to replace:
- source control platforms,
- enterprise project suites,
- or BI/data warehouse products.

It does aim to integrate with existing tools and channels while keeping the agent as the primary interface.

## 5) Core product pillars

### A. Agentic execution
The system can run model responses with tool calls, approval gates, and workflow coordination.

### B. Persistent memory and knowledge
Conversation context can be compressed/summarized, extracted into memory artifacts, and linked to vault knowledge.

### C. Automation and planning
Schedulers and workflow runners allow recurring or multi-step execution beyond single chat turns.

### D. Multi-channel interaction
The same core agent runtime is exposed via web sessions and channel integrations (notably Telegram).

### E. Controlled autonomy
Trust levels and approval flows provide guardrails for destructive/sensitive operations.

## 6) Strategic outcome

When these pillars are combined, Tequila’s intended outcome is an assistant that becomes more useful over time, can act without repeated prompting, and remains understandable and controllable by the user.
